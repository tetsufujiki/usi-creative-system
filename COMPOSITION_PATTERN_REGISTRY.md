# COMPOSITION PATTERN REGISTRY

## Purpose

This file records the current reusable Composition Patterns of the United Studio Creative System.

It is a companion to 06 — COMPOSITION PATTERNS.

The Registry defines reusable relationships.

It does not define:

- complete pages.
- exact layouts.
- framework code.
- CSS.
- token values.
- project-specific content.
- mandatory Pattern order.

## STATUS

Each Pattern uses one of the following statuses.

### EXPERIMENTAL

The relationship is meaningful, but its boundaries or use across real projects remain under evaluation.

### STABLE

The Intent, Moment relationship, Mobile behavior, and Completion Condition have been validated through real composition.

### DEPRECATED

The Pattern should not be used in new work.

A replacement or reason for removal must be documented.

## ENTRY FORMAT

Each Pattern entry should contain:

```
Name
Status
Intent
Starting State
Perceptual Change
Moment Sequence
Dominant Moment
Supporting Roles
May Use
Must Not
Mobile
Completion Condition
Common Misinterpretation
Notes
```

# CORE PATTERNS

## Entry

**Status:** Experimental

**Intent**

Opens the experience and establishes enough orientation for attention to begin.

**Starting State**

- The person has little or no context.
- The tone and meaning of the experience are not yet clear.

**Perceptual Change**

```
from arrival
to orientation and readiness.
```

**Moment Sequence**

```
presence
→ recognition
→ opening
```

**Dominant Moment**

- Recognition of the experience’s central presence.

**Supporting Roles**

- Site identity.
- short contextual language.
- restrained Navigation.
- atmosphere.
- initial Creative Phenomenon.

**May Use**

- Section.
- Frame.
- Heading Group.
- Media.
- Site Header.
- Phenomenon Surface.
- Continuation Link.

**Must Not**

- explain the entire page.
- contain several equal statements.
- introduce unrelated promotional urgency.
- assume Entry requires a full-screen Hero.
- delay meaning only for visual effect.

**Mobile**

- The first relevant viewport must reveal one center.
- Orientation must not depend on Desktop composition.
- Primary meaning should appear before secondary detail.

**Completion Condition**

The person understands what kind of experience has begun and is ready to continue.

**Common Misinterpretation**

- Hero Section.
- splash screen.
- oversized branding.
- cinematic delay before content.

## Statement

**Status:** Experimental

**Intent**

Gives clear form to an important belief, position, or meaning.

**Starting State**

- The meaning is present but not yet articulated.
- The person needs a clear interpretive center.

**Perceptual Change**

```
from diffuse meaning
to a clear position.
```

**Moment Sequence**

```
attention
→ articulation
→ holding
```

**Dominant Moment**

- The statement becoming clear.

**Supporting Roles**

- context.
- restrained evidence.
- Space.
- visual emphasis.
- Creative Phenomenon that carries the same meaning.

**May Use**

- Heading Group.
- Prose.
- Quote.
- Frame.
- Stack.
- Media.
- Phenomenon Surface.

**Must Not**

- enlarge ordinary marketing language and treat it as a principle.
- combine several unrelated statements.
- use Typography alone as a substitute for meaning.
- end without support or Release when the claim requires evidence.

**Mobile**

- The central statement must remain understandable when wrapped.
- Supporting content should follow, not compete.

**Completion Condition**

The person can state what the experience believes or means.

**Common Misinterpretation**

- large heading.
- promotional slogan.
- manifesto fragment without context.

## Environment

**Status:** Experimental

**Intent**

Makes a physical, human, or material reality tangible and trustworthy.

**Starting State**

- The environment is unfamiliar.
- Trust has not yet been grounded in visible reality.

**Perceptual Change**

```
from abstract place
to tangible creative environment.
```

**Moment Sequence**

```
presence
→ evidence
→ context
→ possibility
```

**Dominant Moment**

- The environment becoming real.

**Supporting Roles**

- Caption.
- short contextual Copy.
- people.
- tools.
- material.
- activity.
- restrained Creative Phenomena.

**May Use**

- Media.
- Caption.
- Heading Group.
- Prose.
- Split.
- Stack.
- Phenomenon Trace.

**Must Not**

- turn real Photography into generic decoration.
- use Creative Phenomena that reduce trust.
- explain everything visible in the image.
- present equipment without creative context.
- create a gallery without a central role.

**Mobile**

- Important visual evidence must remain legible.
- Crop must preserve meaning.
- Context should remain connected to Media.

**Completion Condition**

The place, people, or material conditions feel real and understandable.

**Common Misinterpretation**

- photo gallery.
- facility list.
- equipment showcase.
- background image section.

## Narrative

**Status:** Experimental

**Intent**

Develops meaning through perceptible sequence and cause.

**Starting State**

- The result, idea, or belief is visible, but its development is not.

**Perceptual Change**

```
from isolated information
to understood progression.
```

**Moment Sequence**

```
origin
→ development
→ transformation
→ consequence
```

**Dominant Moment**

- The central change or relationship within the story.

**Supporting Roles**

- chronological context.
- Photography.
- work evidence.
- quotations.
- traces.
- transitions.
- reflective pauses.

**May Use**

- Prose.
- Heading Group.
- Media.
- Quote.
- Stack.
- Split.
- Phenomenon Surface.
- Phenomenon Trace.

**Must Not**

- become chronology without meaning.
- give every event equal emphasis.
- repeat the same conclusion after each Moment.
- add motion only to simulate progress.
- become long-form text by default.

**Mobile**

- Sequence must remain clear through reading order.
- Each viewport should preserve one center.
- Transitions must not depend on simultaneous Desktop visibility.

**Completion Condition**

The person understands how one meaningful state became another.

**Common Misinterpretation**

- timeline.
- long article.
- process list.
- scroll animation sequence.

## Work

**Status:** Experimental

**Intent**

Presents a work, project, or body of creation with identity and context.

**Starting State**

- The work is unknown, isolated, or insufficiently understood.

**Perceptual Change**

```
from seeing an item
to understanding a specific creative work.
```

**Moment Sequence**

```
identity
→ encounter
→ context
→ continuation
```

**Dominant Moment**

- The work becoming specific and memorable.

**Supporting Roles**

- creator.
- process.
- year.
- collaborator.
- place.
- category.
- relationship to other works.
- destination for deeper exploration.

**May Use**

- Work Card.
- Media.
- Heading Group.
- Caption.
- Prose.
- Grid.
- Stack.
- Continuation Link.

**Must Not**

- treat every work as equally important by default.
- compress a complete project story into a card.
- create a Grid when sequence is more meaningful.
- use metadata as the Primary experience.
- hide work identity behind decorative interaction.

**Mobile**

- Work identity must appear before secondary metadata.
- Media and title must remain clearly related.
- Linked surfaces require clear touch behavior.

**Completion Condition**

The person can distinguish the work and understand why it belongs in the experience.

**Common Misinterpretation**

- portfolio Grid.
- card list.
- thumbnail gallery.
- case-study template.

## Invitation

**Status:** Experimental

**Intent**

Reveals a meaningful next possibility that follows from the current experience.

**Starting State**

- Meaning has been established.
- A relevant continuation exists, but has not yet been revealed.

**Perceptual Change**

```
from receiving meaning
to recognizing a possible next step.
```

**Moment Sequence**

```
relevance
→ possibility
→ continuation
```

**Dominant Moment**

- The next possibility becoming appropriate and clear.

**Supporting Roles**

- short contextual Copy.
- destination.
- relationship to preceding content.
- restrained Navigation.
- evidence of what continuing may offer.

**May Use**

- Heading Group.
- Prose.
- Continuation Link.
- Section.
- Frame.
- Media.
- Stack.

**Must Not**

- create false urgency.
- interrupt with unrelated conversion language.
- use vague action labels.
- become visually stronger than the experience that justifies it.
- require a button when a Link is sufficient.

**Mobile**

- The continuation must have a clear label and touch target.
- Supporting context should remain concise.
- It must not displace the preceding Moment before it settles.

**Completion Condition**

The person understands what may happen next and why it is relevant.

**Common Misinterpretation**

- CTA block.
- contact banner.
- conversion panel.
- promotional button.

## Release

**Status:** Experimental

**Intent**

Allows a Moment, Pattern, or page to settle at an appropriate level of quiet.

**Starting State**

- The central meaning has been expressed.
- Intensity or narrative movement remains unresolved.

**Perceptual Change**

```
from active attention
to retained meaning.
```

**Moment Sequence**

```
settlement
→ trace
→ quiet
```

**Dominant Moment**

- Meaning becoming stable after expression.

**Supporting Roles**

- final sentence.
- Space.
- still Media.
- Phenomenon Trace.
- ownership context.
- quiet continuation.
- Site Footer.

**May Use**

- Prose.
- Media.
- Caption.
- Phenomenon Trace.
- Continuation Link.
- Site Footer.
- Section.
- Frame.

**Must Not**

- introduce a new unrelated climax.
- repeat the entire page conclusion.
- add urgency only because the page is ending.
- remain visually active without purpose.
- treat Footer content as the Release automatically.

**Mobile**

- The ending should remain perceptible.
- Navigation and ownership information should follow a clear sequence.
- Any remaining phenomenon must settle.

**Completion Condition**

The experience feels complete enough for its meaning to remain without further emphasis.

**Common Misinterpretation**

- Footer.
- final CTA.
- empty Space.
- fade-out animation.

# REGISTRY RULES

1. New Patterns begin as Experimental.
2. Stable status requires validation across real compositions.
3. Pattern names describe relationships, not visual layouts.
4. A Pattern must define a perceptual change.
5. A repeated section does not automatically become a Pattern.
6. A fixed layout belongs to project implementation, not the Core Registry.
7. Project-specific Patterns remain local until their relationship proves broadly reusable.
8. Pattern growth should remain slower than page growth.
9. A Pattern without a clear Completion Condition should not enter the Registry.
10. When uncertain, preserve the relationship locally and do not add a Core Pattern.
