# 06 — COMPOSITION PATTERNS

## Reusable Relationships, Not Repeated Pages

**A Pattern is not a reusable layout.**

**It is a reusable relationship between Moments.**

A Composition Pattern describes how meaning may begin, develop, become tangible, continue, and settle.

It does not prescribe one page structure.

It does not determine visual form.

It preserves a meaningful relationship while allowing each experience to remain distinct.

## 06.0 — PURPOSE

This chapter translates Experience Composition and the Component Library into reusable relationships across a page.

04 — EXPERIENCE COMPOSITION defines:

- the Dominant Moment.
- attention and rhythm.
- continuity.
- restraint.
- the role of expression.

05 — COMPONENT LIBRARY defines:

- reusable Components.
- Component responsibilities.
- Component Anatomy.
- Variants.
- extension boundaries.

06 — COMPOSITION PATTERNS defines:

- how Moments relate.
- how one Moment prepares for another.
- how Components support a shared narrative role.
- how a page gains rhythm without repeating one layout.
- how familiar structures remain specific to each experience.

A Pattern exists between principles and pages.

It is more specific than a principle.

It is less specific than a finished composition.

### PATTERNS ARE NOT TEMPLATES

A Template reproduces structure.

A Pattern preserves relationship.

A Template may say:

```
Heading
Image
Copy
Button
```

A Pattern asks:

```
What should become perceptible first?
What should become trustworthy?
What should deepen the meaning?
What should remain afterward?
```

The same Pattern may appear through:

- Photography.
- Copy.
- Creative Phenomena.
- Typography.
- Space.
- Navigation.
- combinations of Components.

Its structure may change.

Its relationship must remain understandable.

### CHAPTER BOUNDARY

This chapter does not define:

- exact page layouts.
- fixed section order.
- route structure.
- CSS Grid or Flexbox rules.
- component props.
- token values.
- animation duration.
- service-specific workflows.
- conversion funnels.
- product interaction states.
- mandatory calls to action.

Those decisions belong to:

- project composition.
- Implementation Contract.
- project-specific extensions.
- Interaction Extension when needed.

### PRIMARY CONTEXT

The Core Composition Patterns primarily support:

- brand sites.
- studio and environment presentation.
- project and work archives.
- creative philosophy.
- service explanation.
- editorial storytelling.
- photography-led experiences.
- Creative Phenomena.
- long-form scrolling pages.

They may support product interfaces, but they do not define workflow behavior.

## 06.1 — PATTERN AS RELATIONSHIP

A Composition Pattern is defined by the relationship it preserves.

It is not defined by:

- a visual arrangement.
- a section count.
- a background treatment.
- a card style.
- a grid.
- a specific animation.
- a desktop composition.

A valid Pattern should explain:

1. What the person understands or feels before it begins.
2. What change occurs through the Pattern.
3. Which Moments are involved.
4. How attention moves.
5. What kind of evidence or expression is needed.
6. How the Pattern concludes.
7. What it prepares for next.

### THE PATTERN CONTRACT

Every Composition Pattern contains:

- Intent.
- Starting State.
- Perceptual Change.
- Moment Sequence.
- Dominant Moment.
- Attention Path.
- Component Roles.
- Expression Boundary.
- Mobile Behavior.
- Completion Condition.
- Common Misinterpretation.

### INTENT

Intent defines why the Pattern is used.

For example:

- establish the identity of an experience.
- connect an abstract belief to physical reality.
- reveal the process behind a work.
- make several works understandable as one body.
- invite further exploration without breaking tone.
- allow the experience to end with Release.

Intent must be expressed independently of layout.

### STARTING STATE

Starting State describes what is currently unknown, unfelt, or unstable.

Examples include:

- the place is unfamiliar.
- the work is visible, but its origin is not.
- the philosophy is abstract.
- several projects appear unrelated.
- the person has received enough meaning but does not yet know how to continue.
- the experience has reached its final statement but has not yet settled.

### PERCEPTUAL CHANGE

A Pattern must define the change it creates.

A Pattern’s Perceptual Change
is the combined change created
through the sequence of its Moments.

It does not replace
the individual perceptual change
defined for each Moment.

Use the following form:

```
Through this Pattern,
the person moves from ______
to ______.
```

For example:

```
from seeing a studio as a facility
to sensing it as an environment where creation can begin.
```

```
from viewing separate works
to understanding the creative thread that connects them.
```

```
from receiving a strong statement
to holding its meaning in a quieter state.
```

### MOMENT SEQUENCE

A Pattern may contain one or several Moments.

Each Moment must have one center.

The sequence should describe meaning, not Components.

For example:

```
presence
→ context
→ evidence
→ continuation
```

not:

```
Hero
→ two-column section
→ card grid
→ button
```

Components realize the sequence.

They do not define it.

### ATTENTION PATH

Attention within a Pattern should remain directional.

A common structure is:

```
entry
→ recognition
→ deepening
→ release
```

Different Patterns may use different paths.

However, they should not present several equal centers without a meaningful reason.

### COMPONENT ROLES

A Pattern may recommend Component roles, but it should not require one exact Component composition.

For example, an Environment Pattern may use:

- Media as Primary evidence.
- Heading Group as contextual framing.
- Caption as Supporting detail.
- Phenomenon Trace as a restrained continuation.

Another Environment Pattern may use:

- Prose as the Anchor.
- Media as later evidence.
- Space as the primary rhythm.
- no Expressive Component.

The Pattern survives when the relationship remains clear.

### EXPRESSION BOUNDARY

Every Pattern must define how far expression may go.

A Pattern should state:

- which Moment may be strongest.
- which Moment should remain quiet.
- whether Creative Phenomena are appropriate.
- whether Photography is evidence or atmosphere.
- where motion should settle.
- what must remain when expression is reduced.

### COMPLETION CONDITION

A Pattern is complete when its intended perceptual change has occurred.

It is not complete because every planned Component was displayed.

A Pattern should end before another Pattern becomes dominant.

## 06.2 — PATTERN ANATOMY

A Composition Pattern may be represented as:

```
Pattern
├── Entry Condition
├── Dominant Moment
├── Supporting Moment
├── Resolution
└── Continuation
```

Not every Pattern requires every part.

The Anatomy describes functions, not sections.

### ENTRY CONDITION

The Entry Condition defines what the Pattern receives from what came before.

It may receive:

- no previous context.
- a question.
- a strong statement.
- an abstract idea.
- physical evidence.
- a completed project.
- a trace.
- a quiet state.

A Pattern should not ignore the meaning already established.

### DOMINANT MOMENT

The Dominant Moment creates the Pattern’s central change.

It should not share equal dominance with another Moment.

The strongest Photography, Typography, or Creative Phenomenon should usually belong here.

### SUPPORTING MOMENT

A Supporting Moment may:

- clarify.
- prove.
- contextualize.
- humanize.
- slow the rhythm.
- connect abstract meaning to reality.
- connect one work to another.

Supporting Moments must deepen the Pattern.

They should not begin a competing Pattern.

### RESOLUTION

Resolution allows the central meaning to settle.

It may take the form of:

- Space.
- short Copy.
- a still image.
- a Caption.
- a Phenomenon Trace.
- a reduced visual state.
- a quiet compositional change.

Resolution is not necessarily an ending.

It may prepare the next Pattern.

### CONTINUATION

Continuation reveals what may follow.

It may be:

- another Pattern.
- a related work.
- a deeper explanation.
- a different perspective.
- a quiet Navigation path.
- the end of the page.

Continuation should feel caused by what preceded it.

It should not behave like an unrelated promotion.

### MINIMUM VIABLE PATTERN

A Pattern may be very small.

For example:

```
statement
→ evidence
```

or:

```
work
→ context
```

or:

```
movement
→ trace
```

A Pattern should not be expanded only to appear substantial.

### PATTERN SCALE

A Pattern may exist at several scales.

It may describe:

- a relationship within one viewport.
- a relationship between two sections.
- a sequence across several scroll intervals.
- the primary arc of a complete page.

Its scale must be stated clearly.

A page-level Pattern should not be mistaken for a Component composition.

A small Pattern should not be forced to carry the entire page.

## 06.3 — CORE COMPOSITION PATTERNS

This chapter defines
the semantic role of each Core Pattern.

COMPOSITION_PATTERN_REGISTRY.md
is the source of truth
for each Pattern’s current Status,
operational fields,
and validated contract.

Capitalized Entry,
Statement,
and Release
are Pattern names.

Lowercase entry,
statement,
and release
refer to phases
defined by their immediate context.

The Core System begins with seven Composition Patterns.

```
Entry
Statement
Environment
Narrative
Work
Invitation
Release
```

These are not mandatory page sections.

They are recurring roles in the unfolding of meaning.

### ENTRY

Entry establishes orientation and opens the experience.

It answers:

- Where am I?
- What kind of experience is beginning?
- What should I notice first?
- What tone should I enter?

Entry does not need to explain everything.

Its purpose is to establish enough clarity for attention to begin.

A valid Entry may be:

- a strong sentence.
- a photograph.
- a quiet spatial composition.
- a Creative Phenomenon.
- a site identity with restrained context.
- a work shown before explanation.

Entry must not become a compressed summary of the entire page.

### STATEMENT

Statement gives clear form to an important belief, position, or meaning.

It may use:

- Heading Group.
- Prose.
- Quote.
- Typography.
- Space.
- restrained Media.
- one focused Creative Phenomenon.

Statement should not become ordinary promotional language enlarged visually.

It requires language or expression that can sustain emphasis.

A Statement Pattern should be followed by evidence, development, or a Release Pattern.

Repeated Statements without support reduce credibility.

### ENVIRONMENT

Environment makes a place, material, person, or working reality tangible.

Its central responsibility is trust.

Photography often carries the Primary role.

Copy should provide context without explaining what the image already proves.

Creative Phenomena may reveal:

- atmosphere.
- relation.
- trace.
- creative presence.

They must not make the physical environment less trustworthy.

Environment may move from:

```
unfamiliar
→ tangible
→ trustworthy
```

or:

```
place
→ activity
→ possibility
```

### NARRATIVE

Narrative develops meaning through sequence.

It may reveal:

- how an idea began.
- how an 音のタネ grew.
- how dialogue changed a work.
- how one state became another.
- how philosophy appears in practice.
- how a project unfolded.

Narrative is not a long block of text by default.

It is an ordered relationship between Moments.

A Narrative Pattern should preserve:

- causality.
- progression.
- rhythm.
- selective detail.
- a clear ending.

Narrative must not become chronology without meaning.

### WORK

Work presents a project, piece, performance, or body of creative output.

It should establish:

- identity.
- specificity.
- context.
- evidence.
- relationship to the larger Creative System.

A Work Pattern may focus on:

- one work.
- a featured group.
- a contrast between works.
- a continuing archive.
- one project’s process.
- the relationship between result and origin.

Work must not reduce every project to identical cards when hierarchy matters.

The finished work may be Primary.

The process, people, or meaning may also be Primary when the composition requires it.

### INVITATION

Invitation reveals a meaningful next possibility.

It does not create urgency by default.

It may invite a person to:

- explore another work.
- understand a service.
- visit a place.
- begin a conversation.
- consider creating.
- move to a related experience.

Invitation should emerge from the meaning already established.

It should not interrupt with unrelated conversion language.

A valid Invitation answers:

- Why is this continuation relevant now?
- What will become possible by continuing?
- Does the tone remain consistent?
- Is the destination clear?

### RELEASE

Release allows the experience to settle.

It may close:

- a Moment.
- a Pattern.
- a page.
- a sequence of expressive intensity.

Release does not mean absence.

It may preserve:

- a trace.
- a final sentence.
- a still image.
- a quiet continuation.
- ownership context.
- a feeling of possibility.

Release should not introduce a new unrelated argument.

It completes the current experience at an appropriate level of quiet.

## 06.4 — PATTERN SEQUENCES

A page may use several Patterns.

The sequence should emerge from meaning, not from a standard website formula.

Possible sequences include:

```
Entry
→ Environment
→ Invitation
→ Release
```

```
Entry
→ Statement
→ Narrative
→ Work
→ Release
```

```
Work
→ Narrative
→ Environment
→ Invitation
```

```
Statement
→ Environment
→ Work
→ Release
```

These are examples, not templates.

### SEQUENCE BY CAUSE

Each Pattern should create a reason for the next Pattern to appear.

For example:

- Statement creates a need for evidence.
- Environment makes that evidence physical.
- Work shows what became possible there.
- Invitation reveals how the person may continue.
- Release allows the meaning to remain.

A Pattern sequence should be explainable as a progression of meaning.

### DO NOT REPEAT FUNCTION

Several Patterns may use similar Components.

This does not mean they perform the same function.

A large photograph may serve:

- Entry as orientation.
- Environment as evidence.
- Work as the finished result.
- Release as a remaining trace.

The Component may repeat.

The compositional role should remain distinct.

### INTENSITY ACROSS PATTERNS

Patterns should not all use the same intensity.

A page may use:

```
quiet Entry
→ strong Statement
→ grounded Environment
→ reflective Narrative
→ focused Work
→ restrained Invitation
→ Release
```

The exact sequence may vary.

What matters is that emphasis remains selective.

### PATTERN TRANSITIONS

A transition between Patterns should preserve:

- meaning.
- attention.
- rhythm.
- perceptible cause.

Transition may be created through:

- repeated language.
- changing media.
- a shared visual trace.
- contrast.
- continuation of Space.
- a shift from abstraction to reality.
- a shift from result to process.
- a shift from observation to possibility.

A transition should not exist only to demonstrate animation.

### PATTERN OVERLAP

Two Patterns may overlap when one resolves while another begins.

For example:

- Environment may resolve into Narrative.
- Work may become an Invitation.
- Statement may begin within Entry.
- Release may contain a quiet continuation.

Overlap is valid when the Dominant Moment remains clear.

If two Patterns compete equally, their boundary is unclear.

### PATTERN REPETITION

A Pattern may appear more than once on a page.

Repeated use must create development.

For example:

- several Work Patterns may reveal different scales of practice.
- several Narrative Patterns may show different phases.
- several Environment Patterns may reveal place, people, and process.

Repeated Patterns should not repeat the same intensity, composition, or explanation.

## 06.5 — ADAPTATION AND VARIATION

A Pattern should adapt without losing its relationship.

Variation may occur through:

- different Components.
- different media.
- different scale.
- different rhythm.
- different sequence length.
- different degree of expression.
- different content density.

### PRESERVE RELATIONSHIP

When adapting a Pattern, preserve:

- Intent.
- Perceptual Change.
- Moment relationship.
- Completion Condition.

Visual structure may change.

Component selection may change.

The Pattern remains valid when the same meaningful relationship survives.

### MOBILE COMPOSITION

Mobile is not a simplified Pattern.

It reveals the Pattern through sequence.

On Mobile:

- simultaneous relationships often become temporal.
- Supporting content follows Primary meaning.
- large visual fields become focused Moments.
- Pattern boundaries become more perceptible.
- unnecessary repetition becomes more obvious.

Each Pattern should define:

- the first perceptible center.
- the order of Moments.
- which media may be reduced.
- which relationship must remain.
- where the Pattern ends.

### DESKTOP COMPOSITION

Desktop may show more relationships at once.

This does not permit several competing centers.

Desktop may use:

- spatial contrast.
- simultaneous evidence.
- wider Photography.
- asymmetric relationships.
- extended Phenomenon Surfaces.

Desktop should extend the Pattern.

It should not create a different meaning from Mobile.

### CONTENT-DRIVEN VARIATION

Patterns must tolerate real content.

They should support:

- short and long Copy.
- Japanese and English.
- one or several works.
- portrait and landscape Media.
- strong and quiet Photography.
- presence or absence of Creative Phenomena.

Content should influence composition.

The Pattern should not force content into an unsuitable structure.

### PROJECT-SPECIFIC PATTERNS

A project may define Patterns outside the Core Registry.

A project-specific Pattern should remain local when:

- it depends on one service.
- it describes a workflow.
- it relies on specific data.
- it exists only for one campaign.
- its meaning cannot be separated from one project.
- it has not been validated across experiences.

Project use alone does not justify Core status.

## 06.6 — EVALUATION

Every proposed or changed Pattern should answer:

1. What relationship does the Pattern preserve?
2. What is the Starting State?
3. What perceptual change occurs?
4. Which Moments are involved?
5. Which Moment is Dominant?
6. How does attention move?
7. What causes the next Moment or Pattern?
8. Which Components may support it?
9. Which expression should remain restrained?
10. How does it behave on Mobile?
11. What is its Completion Condition?
12. Is it a Pattern, a Component composition, a page, or a Template?
13. Does it remain valid with different visual forms?
14. Would removing the Pattern make the experience clearer?

**If only the layout can be described, the Pattern is not ready.**

**If the relationship cannot survive another visual form, it is a Template.**

## CHAPTER DECISION RULES

The following rules are normative.

### MUST

1. Every Pattern MUST define one Intent.
2. Every Pattern MUST define a perceptual change.
3. Every Pattern MUST identify its Dominant Moment.
4. Every Pattern MUST preserve a meaningful relationship between Moments.
5. Every Pattern MUST define a Completion Condition.
6. Pattern adaptation MUST preserve Intent and relationship.
7. Mobile MUST preserve the semantic sequence of the Pattern.
8. A Pattern MUST NOT require one fixed visual layout.
9. A Pattern MUST NOT determine framework implementation.
10. A Pattern without a distinct reusable relationship MUST NOT enter the Core Registry.

### SHOULD

1. Pattern sequences SHOULD be connected by perceptible cause.
2. Strong Patterns SHOULD be followed by quieter Patterns.
3. Components SHOULD be selected according to role.
4. Pattern transitions SHOULD preserve continuity.
5. Repeated Patterns SHOULD create development.
6. Creative Phenomena SHOULD remain optional.
7. Patterns SHOULD tolerate real content and varied Media.
8. A Pattern SHOULD end before another becomes dominant.

### MAY

1. A Pattern MAY contain one or several Moments.
2. A Pattern MAY span several viewports.
3. A Pattern MAY overlap with an adjacent Pattern.
4. A Pattern MAY use different Component compositions.
5. A Pattern MAY be repeated when its role develops.
6. A project MAY define local Patterns outside the Core Registry.
7. A Pattern MAY be completed without Navigation or Creative Phenomena.

## CODEX INTERPRETATION BOUNDARY

Codex MUST NOT infer that:

- every page requires all Core Patterns.
- Core Patterns define a fixed page order.
- Entry always means a Hero.
- Statement always means large Typography.
- Environment always means a photo gallery.
- Narrative always means long text.
- Work always means a card Grid.
- Invitation always means a prominent button.
- Release always means a Footer.
- every Pattern requires a separate section.
- every Pattern requires animation.
- repeated Patterns should use identical layouts.
- Pattern consistency means visual sameness.
- Pattern names should appear in user-facing code or Copy.
- a page should be expanded until every Pattern is represented.

When composition is unclear, use the following order:

1. Define the page’s intended perceptual change.
2. Identify the necessary Moments.
3. Determine the relationship between Moments.
4. Select the smallest useful Pattern sequence.
5. Assign one Dominant Moment at a time.
6. Select Components by responsibility.
7. Remove repeated or unsupported Patterns.
8. Preserve continuity.
9. End with an appropriate degree of release.
