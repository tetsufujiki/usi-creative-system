# USI Creative System

The United Studio Creative System
defines how creation is understood,
made perceptible,
composed,
reused,
and implemented.

## Core Structure

| Chapter | Responsibility |
|---|---|
| `01_MANIFESTO.md` | Why the Creative System exists and what it promises. |
| `02_CREATIVE_PRINCIPLES.md` | How an 音のタネ grows into creation. |
| `03_CREATIVE_PHENOMENA.md` | How creative principles become perceptible through behavior, time, response, and change. |
| `04_EXPERIENCE_COMPOSITION.md` | How meaning, attention, rhythm, continuity, and restraint become one experience. |
| `05_COMPONENT_LIBRARY.md` | How recurring roles become reusable Component contracts. |
| `06_COMPOSITION_PATTERNS.md` | How relationships between Moments become reusable Composition Patterns. |
| `07_IMPLEMENTATION_CONTRACT.md` | How meaning, role, and relationship survive implementation. |

## Governance

`00_EDITORIAL_RULES.md`
defines how the Creative System is written,
reviewed,
extended,
and maintained.

## Dependency Direction

```text
01 — Purpose and Promise
↓
02 — Creative Meaning
↓
03 — Perceptible Change
↓
04 — Experience Composition
↓
05 — Reusable Roles
↓
06 — Reusable Relationships
↓
07 — Implementation Contract
```

01–04 are the Principle Authority.

05–06 and their Registries
are the Structural Authority.

07 defines how those authorities
are preserved in implementation.

## Core Structure Freeze

The Core chapter structure is fixed as 01–07.

New needs should be handled through
existing chapter revisions,
companion documents,
Creative Fragments,
project-specific extensions,
or the future Interaction Extension.
